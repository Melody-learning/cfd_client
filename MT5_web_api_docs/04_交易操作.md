发送交易请求


该请求可以代表交易员向服务器发送交易请求。


请求格式


> POST /api/dealer/send_request { 交易请求的描述 }


响应格式


> { "retcode" : "code description", "answer" : { "ID" : "request identifier" } }


示例


```
//--- 发送到服务器的请求

POST /api/dealer/send_request

{

 "Action" :"200",

 "Login" :"1010",

 "Symbol" :"EURUSD",

 "Volume" :"100",

 "TypeFill" :"0",

 "Type" :"0",

 "PriceOrder" :"1.11850",

 "Digits" :"5"

}

//--- 服务器响应

{

 "retcode" :"0 Done",

 "answer" : {

 "ID" :"13992"

 }

}
```


请求参数


该请求没有参数。请求描述以JSON格式作为附加主体传递。


- 可能参数的完整描述提供在数据结构部分。

- 必需字段由请求类型来决定。更多详情，请参阅相应的部分。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- id — /api/dealer/get_request_result中用于接收请求执行结果的交易请求标识符。


注意


- 响应代码0并不代表请求的执行。它表示请求已被验证并排队等待服务器处理。

- 响应代码0不表示请求的执行。它表示请求已经过验证并排队等待服务器处理。

- 只有 交易员操作 (200-255) 可以设置为请求类型（操作字段）。

- 通过/api/dealer/get_request_result可以获得交易请求执行结果。

- 最多可同时排队128个交易请求。如果超出此限制，服务器将返回错误10024。

- 每次调用/api/dealer/send_request之后，将自动创建与该请求执行相关的事件订阅。

获取请求执行结果


该请求可以接收服务器上交易请求执行的结果。执行结果以两种形式返回：作为模拟对象[IMTConfirm](imtconfirm.htm)（该变量已经过时但仍被一些交易者使用）和模拟对象[IMTRequest](imtrequest.htm)。


请求格式


> POST /api/dealer/get_request_result?id=identifiers


响应格式


> { "retcode" : "code description", "request identifier" : [ "result" : { result description as an IMTConfirm object }, "answer" : { result description as an IMTResult object } ] }


示例


```
//--- 发送到服务器的请求

GET /api/dealer/get_request_result?id=1,2,3

//--- 服务器响应

{

 "retcode" : "0 Done",

 "1": [

 {

 "result": {

 "ID": "1",

 "DealID": "",

 "OrderID": "",

 ...

 }

 },

 {

 "answer": {

 "ID": "6",

 "IDClient": "1",

 "Login": "1001",

 "SourceLogin": "1000",

 ....

 }

 }

 ],

 "2": [

 {

 "result": {

 "ID": "2",

 "DealID": "",

 "OrderID": "",

 ...

 }

 },

 {

 "answer": {

 "ID": "7",

 "IDClient": "2",

 "Login": "1001",

 "SourceLogin": "1000",

 ...

 }

 }

 ],

 ...

}
```


请求参数


- id — 您希望接收数据的请求的标识符。用逗号隔开指定多个名称。返回请求标识符以响应/api/dealer/send_request。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- id — 交易请求描述数组，其中上级节点是这些请求的标识符。执行结果以两种形式返回：作为模拟对象IMTConfirm（该变量已经过时但仍被一些交易者使用）和模拟对象IMTRequest。参数的描述提供在数据结构部分。如果没有指定"id"参数，则将返回所有活动订阅 的所有事件（结果）列表。


订阅请求事件


每次调用/api/dealer/send_request之后，将自动创建与该请求执行相关的[事件订阅](webapi_dealer_send.htm)。在收到相关请求的最终状态后，订阅将自动删除。如果未在三分钟内选择订阅事件，该订阅将被自动删除以释放已用资源。


如果断开Web API client网络连接（此后必须重新登录），则所有先前创建的订阅都将自动删除。所有订阅仅在通过其发送交易请求的Web API连接的框架中可用。


注意


- 如果没有请求事件，则将返回具有交易请求标识符和空参数列表的节点。例如："3" : []。

- 如果与"id"参数中指定的请求相关的事件订阅不再存在，则相应的结果将不包括在响应中。

Deposit/Withdrawal


该请求可以进行客户账户上的结余操作。


请求格式


```
GET /api/trade/balance?login=login&type=type&balance=sum&comment=comment

POST /api/trade/balance?login=login&type=type&balance=sum&comment=comment
```


响应格式


> { "retcode" : "code description", "answer" : { "Ticket" : "ticket" } }


示例


```
//--- 发送到服务器的请求

GET /api/trade/balance?login=764636&type=2&balance=1000&comment=onlinedeposit

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : { "Ticket" : "136623" }

}
```


请求参数


- login — 应该进行结余操作的账户的登录。

- type — 结余操作的类型。被指定为EnDealAction枚举值：


- 2 — 结余操作。

- 3 — 信贷操作。

- 4 — 其他添加/撤回。

- 5 — 修正操作。

- 6 — 添加奖金。

- balance — 要更改结余的金额。想要添加钱款，请设置正值。想要撤回钱款，请设置负值。

- comment — 对结余操作注释。注释最大长度为 32 个字符（包括行结束符）。如果指定大于该长度的字符串，其会被截取

- check_margin — 如果该参数的值是 1，则在进行结余操作之前检查可用预付款。如果金额撤回大于可用预付款值，则返回错误 10019。如果将参数设置成零，则不检查预付款且将撤回请求的金额，即使其大于可用预付款。如果不传递参数， 则认为启用检查。


响应参数


- retcode — 如果成功，命令返回响应代码 0。否则，其将返回错误代码。

- ticket — 执行结余操作所依据的交易单号。


注意


- 平衡操作进行如下 交易。

- 如果交易类型不正确，则将返回代码3。

- 通过单一操作累计/提取的金额不得超过 1000000000。如果超过该值，则将返回代码4005。

- 撤回的金额不能超过当前可用预付款。如果超过该金额，则将返回代码10019。

- 信贷操作 (TYPE=3) 期间撤回的金额不能超过以前发行的信贷资产的金额。如果超过该金额，则将返回代码10019。

- 任何结余操作期间的提款金额不得超过当前结余。如果超过该金额，则将返回代码10019。

数据结构


交易请求的数据以JSON格式在[/api/dealer/send_request](webapi_dealer_send.htm)和[/api/dealer/get_request_result](webapi_dealer_updates.htm)请求中被传递。


| 方法 | 类型 | 描述 |
| --- | --- | --- |
| ID | 整数 | 请求ID |
| Login | 整数 | 请求中的客户登录名。 |
| ExternalAccount | 字符串 | 外部交易系统中的客户账户。 |
| Group | 字符串 | 发送请求的客户的组。 |
| Symbol | 字符串 | 请求中的交易品种。 |
| Digits | 整数 | 交易请求价格中的小数位数。 |
| Action | 整数 | 交易请求所属操作的类型。传递为EnTradeActions枚举值。 |
| TimeExpiration | 整数 | 交易请求中的过期时间。该日期以自1970年1月1日以来经过的秒数指定。 |
| Type | 整数 | 请求中的订单类型。传递为EnOrderType枚举值。对于结余操作(TA_DEALER_BALANCE)请使用EnDealAction值： DEAL_BALANCE — 结余操作 DEAL_CREDIT — 信贷操作 DEAL_CHARGE — 额外费用 DEAL_CORRECTION — 更正操作 DEAL_BONUS — 添加奖金 DEAL_COMMISSION — 收取手续费 |
| TypeFill | 整数 | 请求中的订单成交类型。传递为EnOrderFilling枚举值。 |
| TypeTime | 整数 | 请求中的订单过期类型。传递为EnOrderTime枚举值。 |
| Flags | 整数 | 其他交易请求标识。传递为EnTradeActionFlags枚举值。 |
| Volume | 整数 | 请求中的操作交易量。一个单位对应于 1/10000 手。 |
| VolumeExt | 整数 | 请求中的操作交易量（准确率提高）。一个单位对应于 1/100000000 手。 |
| Order | 整数 | 交易请求中的订单单号。 |
| OrderExternalID | 整数 | 外部交易系统中的订单ID。 |
| PriceOrder | Float | 交易请求中的订单价格。 |
| PriceTrigger | Float | 触发stop limit订单时提交限价订单所依的价格。 |
| PriceSL | Float | 交易请求中的止损水平。 |
| PriceTP | Float | 交易请求中的止盈水平。 |
| PriceDeviation | 整数 | 执行价格与订单中请求的价格之间的最大允许偏差。 |
| PriceDeviationTop | 整数 | 增加方向的允许价格偏差。 |
| PriceDeviationBottom | 整数 | 减少方向的允许价格偏差。 |
| Comment | 字符串 | 交易请求注释。 |
| ResultRetcode | 字符串 | 交易请求的当前状态。 |
| ResultDealer | 整数 | 处理请求的交易员的登录名。 |
| ResultDeal | 整数 | 由于请求执行而形成的交易的编号。 |
| ResultOrder | 整数 | 由于请求执行而形成的订单的编号。 |
| ResultVolume | 整数 | 交易员为此请求确认的交易量。初始订单交易量。一单位相当于 1/10000 手，例如，10500表示1.05手。 |
| ResultVolumeExt | 整数 | 交易员为此请求确认的交易量（准确度提高）。初始订单交易量。一单位相当于 1/100000000 手，例如，105000000表示1.05手。 |
| ResultPrice | Float | 交易员确认请求所使用的价格。 |
| ResultDealerBid | 整数 | 交易员为此请求确认的卖价。 |
| ResultDealerAsk | 整数 | 交易员为此请求确认的买价。 |
| ResultDealerLast | 整数 | 交易员为此请求确认的最后价。 |
| ResultMarketBid | 整数 | 服务器处理请求时的市场卖价。 |
| ResultMarketAsk | 整数 | 服务器处理请求时的市场买价。 |
| ResultMarketLast | 整数 | 服务器处理请求时的市场最后价。 |
| ResultComment | 字符串 | 确认请求后由交易员添加的注释。 |
| IDClient | 整数 | 已发送请求的客户端的请求ID。 |
| IP | 字符串 | 已发送请求的IP地址。通过Web API发送请求时，会忽略指定的IP值。服务器会将其替换为请求发送的实际地址。设置该属性只对通过Server API发送的请求有意义，因为这些请求全部从服务器上的插件发送。 |
| SourceLogin | 整数 | 执行请求所代表的交易员的登录名。 |
| Position | 整数 | MetaTrader 5平台中的持仓单号（唯一编号）。 |
| PositionBy | 整数 | MetaTrader 5平台中用于关闭操作的持仓单号（唯一编号）。 |
| PositionExternalID | 整数 | 外部交易系统中的持仓单号（唯一编号）。 |
| PositionByExternalID | 整数 | 外部系统中用于关闭操作的持仓单号（唯一编号）。 |

检查订单预付款


该请求可以检查执行该订单所需的预付款是否可用。


请求格式


```
GET /api/trade/check_margin?login=login&symbol=symbol&type=operation&volume=volume&price=price

POST /api/trade/check_margin?login=login&symbol=symbol&type=operation&volume=volume&price=price
```


响应格式


> { "retcode" : "code description", "answer" : { "new": { "Login": "login", ... }, { "current": { "Login": "login", ... } } }


示例


```
//--- 发送到服务器的请求

GET /api/trade/check_margin?login=3018855&symbol=EURUSD&type=1&volume=100000&price=1.27780

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : {

 "new": {

 "Login" : "3018855",

 "CurrencyDigits" : "2",

 "Balance" : "100000.00",

 "Credit" : "0.00",

 "Margin" : "0.88",

 "MarginFree" : "99999.12",

 "MarginLevel" : "11363636.36",

 ...

 },

 {

 "current": {

 "Login" : "3018855",

 "CurrencyDigits" : "2",

 "Balance" : "100000.00",

 "Credit" : "0.00",

 "Margin" : "0.00",

 "MarginFree" : "100000.00",

 "MarginLevel" : "0.00",

 ...

 }

 }

}
```


请求参数


- login — 执行订单的账户登录名。

- symbol — 执行订单的交易品种名称。

- type — 交易订单类型：


- 0 — Buy

- 1 — Sell

- 2 — Buy Limit

- 3 — Sell Limit

- 4 — Buy Stop

- 5 — Sell Stop

- 6 — Buy Stop Limit

- 7 — Sell Stop Limit

- 8 — Close By

- volume — 交易订单的交易量（一单位相当于 1/100000000 手，例如，105000000表示1.05手）。最终订单大小根据指定交易工具的当前合约大小计算。

- price — 交易订单执行类型。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- new — 执行指定订单后的交易账户状态。

- current — 执行指定订单前的交易账户状态。


注意


该检查是在考虑[客户账户](imtaccount.htm)的当前状态（结余、信贷、浮动盈利、未结订单和持仓等）以及客户[组](config_group.htm)的当前市场价格的情况下进行的。

计算持仓盈利


该请求可以计算指定条件下关闭持仓后接收的盈利。


请求格式


```
GET /api/trade/calc_profit?group=group&symbol=symbol&type=operation&volume=volume&price_open=open price&price_close=close price

POST /api/trade/calc_profit?group=group&symbol=symbol&type=operation&volume=volume&price_open=open price&price_close=close price
```


响应格式


> { "retcode" : "code description", "answer" : { "Profit" : "profit", "Profit_rate" : "profit conversion rate" } }


示例


```
//--- 发送到服务器的请求

GET /api/trade/calc_profit?group=demoforex&symbol=EURUSD&type=0&volume=10000000&price_open=1.11500&price_close=1.12500

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : {

 "Profit" : "77.750000",

 "Profit_rate" : "0.777508"

 }

}
```


请求参数


- group — 执行计算的账户组的名称。

- symbol — 执行计算的交易工具的名称。

- type — 持仓方向：买入 - 0，卖出 - 1。

- volume — 持仓交易量（一单位相当于 1/100000000 手，例如，105000000表示1.05手）。

- price_open — 持仓价格。

- price_close — 平仓价格。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- profit — 指定组的入金货币的持仓盈利。

- profit_rate — 将交易品种盈利货币兑换为组入金货币的持仓盈利汇率。


注意


- 利润是使用该组的当前市场价格从交易工具的利润货币转换成组存款货币的。

- 计算中涉及的交易品种一定可用于执行计算的组。

计算买入货币的汇率


该请求可以根据账户组的设置，计算一种货币购买另一种货币的汇率。


请求格式


```
GET /api/trade/calc_rate_buy?base=currency&currency=currency&group=group&symbol=symbol&price=price

POST /api/trade/calc_rate_buy?base=currency&currency=currency&group=group&symbol=symbol&price=price
```


响应格式


> { "retcode" : "code description", "answer" : { "Rate" : "rate" } }


示例


```
//--- 发送到服务器的请求

GET /api/trade/calc_rate_buy?base=EUR&currency=USD

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : { "Rate" : "1.126760" }

}
```


请求参数


- base — 基础货币（已购）的名称。该货币的三个字母的缩写，例如 EUR。所需参数

- currency — 用于支付已购其他货币的货币名称。该货币的三个字母的缩写，例如 USD。所需参数

- group — 执行计算的客户组的名称。如果指定了该组，则使用市场价格计算该兑换率，并考虑该组的点差调整。如果未指定组，则使用市场价格。

- symbol — 计算汇率时应予以考虑的交易品种的名称。如果具有交易品种参数中指定名称的交易品种用于计算兑换率，则'价格'价格将用作该交易品种的价格。如果没有指定交易品种参数，则该交易品种的当前市场价格（包括该客户组的点差调整）将用于兑换。

- price — 交易品种参数中指定的交易品种价格，该价格将用于兑换而非当前市场价格。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- rate — 买入'base'货币以换取'currency'货币的计算出的汇率。

计算卖出货币的汇率


该请求可以根据账户组的设置，计算将一种货币卖出为另一种货币的汇率。


请求格式


```
GET /api/trade/calc_rate_sell?base=currency&currency=currency&group=group&symbol=symbol&price=price

POST /api/trade/calc_rate_sell?base=currency&currency=currency&group=group&symbol=symbol&price=price
```


响应格式


> { "retcode" : "code description", "answer" : { "Rate" : "rate" } }


示例


```
//--- 发送到服务器的请求

GET /api/trade/calc_rate_sell?base=EUR&currency=USD

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : { "Rate" : "1.127860" }

}
```


请求参数


- base — 基础货币（已售）的名称。该货币的三个字母的缩写，例如EUR。

- currency — 用于支付已售货币的货币的名称。该货币的三个字母的缩写，例如USD。

- group — 执行计算的客户组的名称。如果指定了该组，则使用市场价格计算该兑换率，并考虑该组的点差调整。如果未指定组，则使用市场价格。

- symbol — 计算汇率时应予以考虑的交易品种的名称。如果具有交易品种参数中指定名称的交易品种用于计算兑换率，则'价格'价格将用作该交易品种的价格。如果没有指定交易品种参数，则该交易品种的当前市场价格（包括该客户组的点差调整）将用于兑换。

- price — 交易品种参数中指定的交易品种价格，该价格将用于兑换而非当前市场价格。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- rate — 卖出'base'货币以换取'currency'货币的计算出的汇率。
