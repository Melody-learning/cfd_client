<a id="webapi-book-get"></a>


| | Web API / 主接口(Rest API) / 价格 / Get Market Depth | |
| --- | --- | --- |






获取市场深度


该请求可以接收来自交易品种“市场深度”的数据。


请求格式


> GET /book_get?symbol=symbols POST /book_get?symbol=symbols


响应格式


> { "retcode" : "code description", "answer" : [ { Market Depth description }, { Market Depth description }, ... ] }


示例


```
//--- 发送到服务器的请求

/book_get?symbol=EURUSD

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : [

 {

 "Symbol": "EURUSD",

 "Datetime": 1586804133000,

 "Flags": 1,

 "Items": [

 { "Type": 1, "Price": 1.091460, "Volume": 1000 },

 { "Type": 2, "Price": 1.091370, "Volume": 1000 }

 ]

 }

 ]

}
```


请求参数


- symbol — 您想要接收“市场深度”数据的交易品种。您可以用逗号分隔指定多个交易品种，或者使用"*"掩码和否定符号"!"来指定多个交易品种组。例如：


- symbol=EURUSD,USDJPY — 获得交易品种EURUSD和USDJPY的报价。

- symbol=Forex\Major*, GOLD — 获得来自Major子组的所有交易品种的报价以及GOLD的报价。

- symbol=Forex\Crosses*,!AUDUSD — 获得Crosses子组的所有交易品种的报价，除了AUDUSD以外。

- symbol=Forex\Major\EUR* — 通过来自Major子组的基本货币EUR获得所有交易品种的报价。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- answer — “市场深度”价格数组。传递参数的描述提供在“数据结构”部分。


注意


“市场深度”数据只能针对该账户可用的交易品种获得。


代码"1 OK/None"表示所需的数据在访问服务器中不存在，并且需要更多时间从历史服务器接收数据。在这种情况下，您应该在将历史服务器中的数据上传到访问服务器之后，稍后再次请求。

<a id="webapi-chart-get"></a>


| | Web API / 主接口(Rest API) / 价格 / 获取M1历史 | |
| --- | --- | --- |






获取柱形图历史


该请求可以接收柱形图历史（1分钟数据）。


请求格式


```
GET /api/chart/get?symbol=symbol&from=date&to=date&data=data

POST /api/chart/get?symbol=symbol&from=date&to=date&data=data
```


响应格式


> { "retcode" : "code description", "answer" : [ [ bar description ], [ bar description ], ... ] }


示例


```
//--- 发送到服务器的请求

/api/chart/get?symbol=EURUSD&from=1574351000&to=1574351349&data=dohlc

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : [

 [1574351040,1.10779,1.10798,1.10778,1.10798],

 [1574351100,1.10798,1.10807,1.10790,1.10795],

 [1574351160,1.10795,1.10795,1.10775,1.10777],

 ...

 ]

}
```


请求参数


- symbol — 您希望接收的交易品种的1分钟数据。

- from — 希望接收数据的时段的开始。该日期以自 1970 年 1 月 1 日以来经过的秒数指定。

- to — 希望接收数据的时段的结束。该日期以自 1970 年 1 月 1 日以来经过的秒数指定。

- data — 要接收的数据集。它指定为一串字母，每个字母对应于请求数据类型：


- d — 柱形图日期和时间。该日期以自 1970 年 1 月 1 日以来经过的秒数指定。

- o — 开仓价。柱形图开始形成时的价格（分钟开始）。

- h — 最高价。柱形图时段的最高价。

- l — 最低价。柱形图时段的最低价。

- c — 平仓价。柱形图结束时的价格（分钟结束）。

- t — 报价量。柱形图形成()期间接收的报价量。

- v — 真实交易量。柱形图形成期间执行交易的真实交易量。

- s — 点差。柱形图形成期间记录的最低交易品种点差。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- answer — 柱形图数组。数据集取决于请求中的"data"参数。


注意


每个响应参数大小限制为16兆字节。如果匹配查询条件的响应参数较大，则返回响应参数的前16兆字节，以及响应代码“14个操作尚未完成”。在这种情况下，应该重复请求，使用当前结果中最后一行的日期-时间作为新请求的"from"参数的值。应重复此操作，直到收到响应代码"0 Done"。


代码"1 OK/None"表示所需的数据在访问服务器中不存在，并且需要更多时间从历史服务器接收数据。在这种情况下，您应该在将历史服务器中的数据上传到访问服务器之后，稍后再次请求。


执行请求[的经理账户](webapi_rest_authentication.htm#client_start)必须有[IMTConManager::RIGHT_CFG_SYMBOLS](imtconmanager_enum.htm#enmanagerrights)和[IMTConManager::RIGHT_CHARTS](imtconmanager_enum.htm#enmanagerrights)权限。此外，经理所属的组必须有权访问相应的交易品种([IMTConGroup::Symbol*](imtcongroup_symboladd.htm))。如果所需的权限不可用，该方法将返回[13](retcodes_common.htm)错误。

<a id="webapi-symbol-add"></a>


| | Web API / 主接口(Rest API) / 配置数据库 / 交易品种 / 添加 | |
| --- | --- | --- |






添加交易品种


使用这个请求，您可以创建或更改服务器上的交易品种。


请求格式


> POST /api/symbol/add { JSON格式的将要创建的交易品种描述 }


响应格式


> { "retcode" : "code description", "answer" : { Description of the created symbol in JSON format } }


示例


```
//--- 发送到服务器的请求

POST /api/symbol/add

{

 "Symbol" : "EURUSD",

 "Path" : "Forex\\Major\\EURUSD",

 "ISIN" : "",

 "Description" : "Euro vs US Dollar",

...

}

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : {

 "Symbol" : "EURUSD",

 "Path" : "Forex\\Major\\EURUSD",

 "ISIN" : "",

 "Description" : "Euro vs US Dollar",

...

 }

}
```


请求参数


该命令没有参数。交易品种的描述以 JSON 格式作为附加命令主体传递。添加交易品种时，所有其[参数](webapi_symbol_data_structure.htm#symbol)都必须被描述。


创建时传递的交易品种的 JSON 描述与服务器返回的描述相同。


响应参数


- retcode — 如果成功，命令返回响应代码 0。否则，其将返回错误代码。

- answer — JSON格式的已创建交易品种的参数。传递交易品种参数的完整描述提供在“数据结构”部分。


注意


- 该命令仅在被连接至主交易服务器时起作用。否则将返回12001。

- 运行命令时，会检查您正添加的交易品种是否存在。用于比较的键字段是交易品种名称。如果已经存在该等交易品种，则更新其设置。

- 当您更新配置时，仅更改那些在 JSON 描述中明确指定的参数。其他参数保持不变。

- 添加前会检查记录的正确性。如果条目不正确，其返回错误代码 3。

- 若要运行这个命令，经理账户必须有权作为管理员连接以及编辑交易品种配置(IMTConManager::RIGHT_ADMIN，IMTConManager::RIGHT_CFG_SYMBOLS)。否则，其返回错误代码8。

| 选项字段 | 类型 | 描述 |

| Symbol | 字符串 | 交易品种名称。 |
| Path | 字符串 | 交易品种路径。 |
| ISIN | 字符串 | 交易品种的国际证券识别码 (ISIN)。 |
| Description | 字符串 | 交易品种描述。 |
| International | 字符串 | 国际交易品种名称。 |
| Category | 字符串 | 交易品种所属的类别或部分的名称。 |
| Exchange | 字符串 | 进行证券交易的交易所名称。 |
| CFI | 字符串 | 符合ISO 10962标准的交易品种分类。 |
| Sector | 整数 | 交易品种所属的经济部门。传递为EnSectors枚举值。 |
| Industry | 整数 | 交易品种所属的行业分支。传递为EnIndustries枚举值。 |
| Country | 字符串 | 在证券交易所交易的股票的公司所在的国家。 |
| Basis | 字符串 | 衍生金融工具的基础资产。 |
| Source | 字符串 | 报价用于当前金融工具的源交易品种的名称。 |
| Page | 字符串 | 交易品种的网页地址。 |
| CurrencyBase | 字符串 | 交易品种的基础货币。 |
| CurrencyBaseDigits | 整数 | 转换成基础货币的精度。 |
| CurrencyProfit | 字符串 | 交易品种的利润货币。 |

| QuotesTimeout | 整数 | 等待报价的时间，其后自动针对交易品种禁用交易。 |
| VolumeMin | 整数 | 针对交易品种的交易操作的最小交易量。一单位相当于 1/10000 手，例如，10500表示1.05手。 |
| VolumeMax | 整数 | 针对交易品种的交易操作的最大交易量。一单位相当于 1/10000 手，例如，10500表示1.05手。 |
| VolumeStep | 整数 | 针对交易品种的交易操作的量更改幅度。一单位相当于 1/10000 手，例如，10500表示1.05手。 |
| VolumeLimit | 整数 | 单向交易品种允许的持仓和订单的最大累积交易量。一单位相当于 1/10000 手，例如，10500表示1.05手。 |
| MarginFlags | 整数 | 其他预付款检查模式。以EnMarginFlags枚举值传递。 |
| MarginInitial | Float | 初始预付款的大小。 |
| MarginMaintenance | Float | 维持预付款的大小。 |
| MarginInitialBuy | Float | 市场买入订单的初始预付款比率。在 PHP 和 .NET 执行中，系数存储在 MarginRateInitial 数组中，可通过指数 MTEnMarginRateTypes::MARGIN_RATE_BUY 获得。 |
| MarginInitialSell | Float | 市场卖出订单的初始预付款比率。在 PHP 和 .NET 执行中，系数存储在 MarginRateInitial 数组中，可通过指数 MTEnMarginRateTypes::MARGIN_RATE_SELL 获得。 |
| MarginInitialBuyLimit | Float | 限价买入订单的初始预付款比率。在 PHP 和 .NET 执行中，系数存储在 MarginRateInitial 数组中，可通过指数 MTEnMarginRateTypes::MARGIN_RATE_BUY_LIMIT 获得。 |
| MarginInitialSellLimit | Float | 限价卖出订单的初始预付款比率。在 PHP 和 .NET 执行中，系数存储在 MarginRateInitial 数组中，可通过指数 MTEnMarginRateTypes::MARGIN_RATE_SELL_LIMIT 获得。 |

```
//--- 发送到服务器的请求
```

```
 ]

```


请求参数

- to — 希望接收数据的时段的结束。该日期以自 1970 年 1 月 1 日以来经过的秒数指定。

- data — 要接收的数据集。它指定为一串字母，每个字母对应于请求数据类型：

- a — 买价。

- v — 最后交易量。

- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

注意


每个响应参数大小限制为16兆字节。如果匹配查询条件的响应参数较大，则返回响应参数的前16兆字节，以及响应代码“14个操作尚未完成”。在这种情况下，应该重复请求，使用当前结果中最后一行的日期-时间作为新请求的"from"参数的值。应重复此操作，直到收到响应代码"0 Done"。


代码"1 OK/None"表示所需的数据在访问服务器中不存在，并且需要更多时间从历史服务器接收数据。在这种情况下，您应该在将历史服务器中的数据上传到访问服务器之后，稍后再次请求。

请求参数


- symbol — 用逗号隔开的交易品种，应该接收其价格。您可以使用掩码 “*” 以及否定符号 “!” 来指定交易品种组。示例：


- symbol=EURUSD,USDJPY — 获得EURUSD和USDJPY的报价。

- symbol=Forex\Major*, GOLD — 获得来自Major子组的所有交易品种的报价以及GOLD的报价。

- symbol=Forex\Crosses*,!AUDUSD — 获得Crosses子组的所有交易品种的报价，除了AUDUSD以外。

- symbol=Forex\Major\EUR* — 通过来自Major子组的基本货币EUR获得所有交易品种的报价。

- trans_id —交易识别符。第一个请求期间指定了 0，这意味着数据的第一个请求。在这种情况下，服务器传递所有信息。在响应命令中，服务器返回 TRANS_ID 识别符，其对应于传递的数据。通过该 ID 的价格的下一个请求期间，服务器将仅返回自上一个请求以来已更改的数据。该等系统允许播送价格时中断网络流量。


> 仅在指定交易品种或组的全路径时可以使用掩码。例如，您不能指定 “USD*”，正确的变体是 “Forex\Major\USD*”。 您可以省略交易品种的路径规格，只要您输入其全名。例如，USDCAD。


响应参数


- retcode — 如果成功，命令返回响应代码 0。否则，其将返回错误代码。

- trans_id — 已发送统计数据的ID以进一步请求更改。

- answer — JSON格式交易品种的统计数据数组。传递统计数据的完整描述提供在“数据结构”部分。
