身份验证

要向服务器发送命令，客户应在那里通过身份验证。这两个命令用于：
•/auth_start — 身份验证开始命令。
•/auth_answer — 身份验证请求响应命令。

身份验证分为几个步骤，借此服务器和客户通过不同参数发送这些命令。
•开始身份验证的请求
•开始在访问服务器上进行身份验证
•响应服务器
•身份验证结束

JavaScript和PHP的身份验证示例提供在示例部分。

开始身份验证的请求

客户应该发送请求以使用/auth_start命令开始验证，例如：



GET https://server.address:443/auth_start?version=484&agent=test&login=14&type=manager
 

在命令中传递以下用"&"字符隔开的参数：
•version — 网页客户端的版本。使用这个字段来管理您的应用程序版本。当网页客户端连接到平台时，指定的版本将被记录到访问服务器日志。例如，"1000"：登录名(Manager WebAPI build 3470)。为了方便起见，您可以在这里指定当前的平台版本。如果您不需要版本管理，可以输入任何不小于484的数字。
•agent — Web代理的任意名字。名字不应为空。名称不应为空且不应包含空格。
•login — 应该根据该账户的登录名执行验证。
•type — 账户类型。目前只支持一个值，就是'manager'。这意味着使用经理账户连接。使用的账户完全决定连接权限，包括允许连接的IP地址列表。



验证开始请求(/api/auth/start)和向服务器发送响应(/api/auth/answer)之间的时间不应超过10秒。否则，身份验证将失败。
 

开始在访问服务器上进行身份验证

响应身份验证开始命令时，访问服务器发送包含响应代码和随机序列的响应。



{ 
  "retcode" : "0 Done", 
  "version_access" : "1290", 
  "srv_rand"       : "d4e005317e38bc0c349a51a0d73a07eb" 
}
 

在命令中传递以下用"|"字符隔开的参数：
•retcode — 响应代码及其字符串描述。0 意味着成功完成。
•version_access — 用于连接至交易服务器的访问服务器的版本；
•srv_rand — 由服务器生成的Hex格式的随机16 字节序列。发送响应时该序列应该应用密码。

响应服务器

接收随机序列之后，客户应该向其应用密码（密码散列）并使用/auth_answer命令将其发送到服务器：



GET https://server.address:443/auth_answer?srv_rand_answer=8b67609a264764a528ec8c709ab71df2&cli_rand=34b6bccab8bfebdf462d1312043df1cc
 

在命令中传递以下用"&"字符隔开的参数：
•srv_rand_answer — MD5随机序列散列和密码散列。
•cli_rand —  您应该自己生成一个用于服务器身份验证的随机序列。序列应该是16 字节大小的Hex格式。

密码散列

密码散列计算方式如下：



Password hash=MD5(MD5('Password')+'WebAPI')
 

WebAPI 的六个字符被添加到 MD5 密码散列中，并再次从结果获得 MD5 散列。然后，发送到服务器时，获得的散列再次结合服务器的随机顺序。散列取自结果值，然后其被发送到服务器。这样可确保授权的最大安全性。

构成srv_rand_answer的示例：



password hash=MD5(MD5('Password1')+'WebAPI')=904ba8ecb16273d2f0ae9c3b8a023752 
srv_rand=73007dc7184747ce0f7c98516ef1c851 
//--- 计算公式：srv_rand_answer=MD5(password hash+srv_rand) 
srv_rand_answer=MD5(904ba8ecb16273d2f0ae9c3b8a023752+73007dc7184747ce0f7c98516ef1c851)=77fe51827f7fa69dd80fbec9aa33f1bb
 

如何构成srv_rand_answer的逐步讲解的示例：

让我们来分析Web password=Password1和srv_rand="73007dc7184747ce0f7c98516ef1c851"。

1.我们用 Unicode (UTF-16-LE)计算密码的MD5。在这种情况下，unicode字符串Password1：



MD5('Password1')
 

结果作为字节：



[ 236,46,146,255,112,191,70,145,174,12,104,224,47,220,108,34 ]
 

2.将单字节WebAPI字符串作为6个字节(UTF-8)添加到字节结果：



MD5('Password1')+'WebAPI'
 

'WebAPI’ 被表现为 = [ 87,101,98,65,80,73 ]。结果作为字节：



[ 236,46,146,255,112,191,70,145,174,12,104,224,47,220,108,34,87,101,98,65,80,73 ]
 

3. 获得密码的最终散列作为来自步骤 2 的字节结果的 MD5：



MD5(MD5('Password1')+'WebAPI')
 

结果作为字节：



[ 144,75,168,236,177,98,115,210,240,174,156,59,138,2,55,82 ]
 

结果作为 HEX 字符串：



904ba8ecb16273d2f0ae9c3b8a023752
 

4。结合来自步骤 3 的密码字节散列以及 SRV_RAND：



MD5(MD5('Password1')+'WebAPI')+ [ 115,0,125,199,24,71,71,206,15,124,152,81,110,241,200,81 ]
 

在此[ 115,0,125,199,24,71,71,206,15,124,152,81,110,241,200,81 ]是转换成字节数组的HEX 字符串73007dc7184747ce0f7c98516ef1c851：



srv_rand = 73007dc7184747ce0f7c98516ef1c851 = [ 115,0,125,199,24,71,71,206,15,124,152,81,110,241,200,81 ]
 

结合的最终结果（两个字节数组被简单地结合起来）：



[ 144,75,168,236,177,98,115,210,240,174,156,59,138,2,55,82,115,0,125,199,24,71,71,206,15,124,152,81,110,241,200,81 ]
 

5. 构成最终SRV_RAND_ANSWER以作为来自步骤4的字节结果的MD5对服务器响应：



MD5(MD5(MD5('Password1')+'WebAPI') + [ 115,0,125,199,24,71,71,206,15,124,152,81,110,241,200,81 ]
 

结果作为字节：



[ 119,254,81,130,127,127,166,157,216,15,190,201,170,51,241,187 ]
 

结果被转换成 HEX 表现：



77fe51827f7fa69dd80fbec9aa33f1bb
 

响应字符串中的最终参数如下：



srv_rand_answer=77fe51827f7fa69dd80fbec9aa33f1bb
 

身份验证结束

通过交易服务器验证客户之后，访问服务器发送响应：



{  
  "retcode" : "0 Done", 
  "version_access"  : "1290", 
  "version_trade"   : "1290", 
  "cli_rand_answer" : "8b67609a264764a528ec8c709ab71df2" 
} 
  
LLLLKKKKFAUTH_ANSWER|RETCODE=0 Done|CLI_RAND_ANSWER=8b67609a264764a528ec8c709ab71df2|CRYPT_RAND=HEX_256_bytes\r\n
 

在命令中传递以下用"|"字符隔开的参数：
•retcode — 响应代码及其字符串描述。0 意味着成功完成。
•version_access — 用于连接至交易服务器的访问服务器的版本；
•version_trade — 交易服务器版本；
•cli_rand_answer — 随机序列的MD5散列和密码散列。序列是Hex格式，大小为16个字节。

为了授权客户服务器，使用了与服务器上客户授权所用相同的算法。在cli_rand_answer参数中，服务器发送客户的随机序列的MD5散列和密码散列。

构成cli_rand_answer的示例：



password hash=MD5(MD5('Password1')+'WebAPI')=904ba8ecb16273d2f0ae9c3b8a023752 
cli_rand=4db98fec17aab4dc5a240bdc659e8395 
//--- 计算公式：cli_rand_answer=MD5(password hash+CLI_RAND) 
cli_rand_answer=MD5(904ba8ecb16273d2f0ae9c3b8a023752+4db98fec17aab4dc5a240bdc659e8395)=1732021b1b332f51660b2b315eed3a35
