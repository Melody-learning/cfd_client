<a id="webapi-attachment-add"></a>


| | Web API / 主接口(Rest API) / 客户 / 添加附件 | |
| --- | --- | --- |






添加附件到数据库


该请求可以向文件数据库添加附加文件。


请求格式


> POST /api/attachment/add [ 一个或多个JSON格式的附件描述 ]


响应格式


> [ { "id" : "identifier", "retcode" : "code description", }, { "id" : "identifier", "retcode" : "code description", }, ... ]


示例


```
//--- 发送到服务器的请求

POST /api/attachment/add

[

 {

 "RelatedClient" : "1032",

 "FileType" : "4",

 "FileName" : "image 2020-03-12 09-53.jpg",

 },

 {

 "RelatedClient" : "1032",

 "FileType" : "2",

 "FileName" : "image 2020-03-12 09-54.jpg

 }

]

//--- 服务器响应

[

 {

 "id" : "730",

 "retcode" : "0 Done",

 },

 {

 "id" : "731",

 "retcode" : "0 Done",

 },

 ...

]
```


请求参数


该请求没有参数。要创建的附件的描述将在请求的JSON主体中传递。


RelatedClient字段为必填字段。可能用户参数的完整描述提供在[“数据结构”](webapi_client_data_structure.htm#attachment)部分。


创建时传递的文档的 JSON 描述与服务器返回的描述相同。例如；


> { "RelatedClient" : "1032", "FileType" : "2", "FileName" : "image 2020-03-12 09-54.jpg ... }


响应参数


- id — 已创建附件的标识符。

- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。


注意


收到附件标识符后，您可以使用[/api/attachment/attach](webapi_attachment_attach.htm)请求将附件添加到文档或请求中。

<a id="webapi-attachment-attach"></a>


| | Web API / 主接口(Rest API) / 客户 / 绑定/解绑附件 | |
| --- | --- | --- |






绑定/解绑附件


该请求可以在评论或文档中附加文件或从评论或文档中删除附件。


请求格式


```
GET /api/attachment/attach?id=attachments&entity=destination&entity_id=identifier&action=action

POST /api/attachment/attach?id=attachments&entity=destination&entity_id=identifier&action=action
```


响应格式


> { "retcode" : "code description", }


示例


```
//--- 发送到服务器的请求

GET /api/attachment/attach?id=731&entity=document&entity_id=41084&action=attach

//--- 服务器响应

{

 "retcode" : "0 Done",

}
```


请求参数


- id — 附件标识符。可以使用Can be received using /api/attachment/add和/api/attachment/get请求进行接收。

- entity — 附加文件的实体（或从这里删除附件）：文档或评论。

- entity_id — 文档或评论标识符。

- action — 要执行的操作：附加或分开。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

<a id="webapi-comment-add"></a>


| | Web API / 主接口(Rest API) / 客户 / 添加注释 | |
| --- | --- | --- |






向文档或客户添加注释


该请求可以向客户记录或文档添加注释。


请求格式


> POST /api/comment/add [ 一个或多个JSON格式的注释描述 ]


响应格式


> [ { "id" : "identifier", "retcode" : "code description", }, { "id" : "identifier", "retcode" : "code description", }, ... ]


示例


```
//--- 发送到服务器的请求

POST /api/comment/add

[

 {

 "RelatedClient" : "1032",

 "Flags" : "0",

 "Text" : "Called client, no answer",

 "CommentType" : "2"

 },

 {

 "RelatedClient" : "1032",

 "Flags" : "0",

 "Text" : "Called client, ready to proceed with registration",

 "CommentType" : "2"

 }

]

//--- 服务器响应

[

 {

 "id" : "1278",

 "retcode" : "0 Done",

 },

 {

 "id" : "1279",

 "retcode" : "0 Done",

 },

 ...

]
```


请求参数


该请求没有参数。要创建的注释的描述将在请求的JSON主体中传递。


必填字段为"RelatedClient"或"RelatedDocument"，基于这些字段，将注释绑定到客户记录或文档。可能用户参数的完整描述提供在[“数据结构”](webapi_client_data_structure.htm#comment)部分。


创建时传递的文档的 JSON 描述与服务器返回的描述相同。例如；


> { "RelatedClient" : "1032", "Flags" : "0", "Text" : "<font face=\"Tahoma\">Called client, no answer<\/font>", "CommentType" : "2" ... }


响应参数


- id — 创建的注释标识符。

- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。


注意


只能从连接到客户已被创建所在的交易服务器的应用程序中添加注释。对于所有其他应用程序，将返回响应代码[12001](retcodes_api.htm)。如果找不到该客户，则将返回响应代码[13](retcodes_common.htm)。

<a id="webapi-comment-get"></a>


| | Web API / 主接口(Rest API) / 客户 / 获取注释 | |
| --- | --- | --- |






获取有关文档或客户的注释


该请求可以接收添加到文档或客户记录的注释。


请求格式


```
GET /api/comment/get?id=list of IDs

GET /api/comment/get?client=list of identifiers&position=position&total=number

GET /api/comment/get?document=list of identifiers&position=position&total=number

POST /api/comment/get?id=list of IDs

POST /api/comment/get?client=list of identifiers&position=position&total=number

POST /api/comment/get?document=list of identifiers&position=position&total=number
```


响应格式


> { "retcode" : "code description", "answer" : [ { comment description }, { comment description }, ... ] }


示例


```
//--- 发送到服务器的请求

GET /api/comment/get?client=1032&position=0&total=2

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : [

 {

 "RecordID" : "1278",

 "RelatedClient" : "1032",

 "Flags" : "0",

 "Text" : "Called client, no answer",

 "CommentType" : "2"

 ...

 },

 {

 "RecordID" : "1279",

 "RelatedClient" : "1032",

 "Flags" : "0",

 "Text" : "Called client, ready to proceed with registration",

 "CommentType" : "2"

 ...

 }

 ]

}
```


请求参数


- id — 您希望获得数据的文档ID。指定为逗号隔开的列表。

- client — 您希望获得注释的客户ID。使用此参数，可以使用参数"position"和"total"对注释进行额外过滤。

- document — 您希望接收注释的文档标识符。使用此参数，可以使用参数"position"和"total"对注释进行额外过滤。

- position — 在客户或文档注释列表中的位置，从这个位置开始获得注释。可选参数，可以与"client"或"document"参数一起使用。它默认等于0（列表中的第一个位置）。

- total — 应该获得的注释的数目。可选参数，可以与"client"或"document"参数一起使用。默认请求所有注释。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- answer — JSON格式注释描述的数组。传递参数的完整描述提供在“数据结构”部分。

| Enable | 整数 | 配置状态。传递为EnFeedersMode枚举值。 |
| Mode | 整数 | 数据源操作模式。传递为EnFeederFlags枚举值。 |
| TimeoutReconnect | 整数 | 重新连接至源服务器的每次尝试之间的超时。 |
| TimeoutSleep | 整数 | 重新连接至源服务器的每个系列之间的超时。 |
| AttemptSleep | 整数 | 重新连接至源服务器的系列中的尝试数目。 |
| State | 数组 | 数据源操作统计数据。 |
| Params | 数组 | 数据源参数。 |
| Symbols | 数组 | 数据源提供报价的交易品种列表。 |
| Translates | 数组 | 转换设置，用于数据源传输的价格数据。 |


数据源操作统计数据


| 参数 | 类型 | 用途 |
| --- | --- | --- |
| SysConnection | 整数 | 数据源到信息源的连接状态：0 — 已连接，1 — 断开连接。 |
| SysLastTime | 整数 | 最近一次数据源成功连接到信息源的时间。 |
| Company | 字符串 | 签署数据源可执行文件的公司。 |
| Issuer | 字符串 | 向上述公司颁发证书的认证机构。 |

| TicksCount | 整数 | 由数据源接收到的当前时段的价格变动次数(MTTick)。 |
| BooksCount | 整数 | 由数据源接收到的当前时段的市场深度变动次数(MTBookDiff)。 |
| NewsCount | 整数 | 由数据源接收到的当前时段的OF新闻条目数(MTNews)。 |
| BytesReceived | 整数 | 在当前交易时间段由数据源接收的流量数。 |
| BytesSent | 整数 | 由数据源发送的当前时段的流量数。 |

| AuthMode | 整数 | 组中账户的授权模式。以EnAuthMode枚举值传递。 |
| AuthPasswordMin | 整数 | 组中账户的最小密码长度。密码最小长度为8个字符，最大长度为16个字符。 |

| CompanySupportPage | 字符串 | 为组提供服务的公司的技术支持网站地址。 |
| CompanySupportEmail | 字符串 | 为组提供服务的公司的技术支持电子邮件地址。 |
| CompanyCatalog | 字符串 | 为公司存储报告和电子邮件等内容的模板的子目录，该公司为组提供服务。该公司为组提供服务。 |

| CurrencyDigits | 整数 | 组入金货币中小数点之后的数字位数。 |
| ReportsMode | 整数 | 报告生成模式。以EnReportsMode枚举值传递。 |
| ReportsFlags | 整数 | 报告发送选项。传递为EnReportsFlags枚举值（适当标识值的和）。 |
| ReportsSMTP | 字符串 | 发送报告的 SMTP 服务器的地址。 |
| ReportsSMTPLogin | 字符串 | 用于发送报告的 SMTP 服务器上的授权登录。 |

响应格式


> { "retcode" : "code description" }


示例


```
//--- 发送到服务器的请求

POST /api/tls_certificate/add

{

 "password": "secret_passrod",

 "cert": "MIISFQIBAzCCEEAQAAADA3MB8wBwYFKw4 ... EOH4gtNlQXeMX/LmfpBcbW5VOJ6BBRgKp+aktx2dQjo1MDRKCfBe67yJQ=="

}

//--- 服务器响应

```


请求参数

- 在所需的'cert'字段，证书二进制主体以Base64的格式传递 — 证书带有PFX格式密钥。如果文件包含多个证书（例如，证书链），则将仅安装第一个证书。

声明


- 该请求仅在被连接至主交易服务器时起作用。否则将返回12001。

- 请求执行期间，将检查是否存在要添加的证书。如果存在，则更新证书。否则，将安装新证书。该检查基于"Thumbprint"证书字段。

- 若要运行这个请求，经理账户必须有权作为管理员连接以及编辑网络配置。否则，将返回错误代码8。

- 为集群内的所有访问服务器安装证书。

- 使用HTTPS协议连接访问服务器所需的证书。这可以将Web API命令作为普通的GET和POST请求发送到服务器。有关详细信息，请访问“通过HTTPS请求”部分。
