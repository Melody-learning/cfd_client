Web API提供了以下请求，用于获得交易平台的常规设置：


| 请求 | 描述 |
| --- | --- |
| /api/common/get | 获取平台常规配置。 |
| /api/common/set | 更新平台常规配置。 |


传递常规配置相关数据的格式，在 [“数据结构”](webapi_common_data_structure.htm)中描述。

<a id="webapi-common-data-structure"></a>


| | Web API / 主接口(Rest API) / 配置数据库 / 常规配置 / 数据结构 | |
| --- | --- | --- |






数据结构


常规配置以JSON格式被传递为响应[/api/common/get](webapi_common.htm)和[/api/common/set](webapi_common_set.htm)请求。常规配置包括以下参数：


| 选项字段 | 类型 | 描述 |
| --- | --- | --- |
| Name | 字符串 | 服务器名称。 |
| NameFull | 字符串 | 服务器全名。 |
| Owner | 字符串 | 平台所有人名称。 |
| OwnerID | 字符串 | 平台所有人短名称。 |
| OwnerHost | 字符串 | 平台所有人主机地址。 |
| OwnerEmail | 字符串 | 平台所有人电子邮件地址。 |
| Product | 字符串 | 产品全名。 |
| ExpirationLicense | 整数 | 许可到期日。 |
| ExpirationSupport | 整数 | 技术支持到期的日期在许可中指定。 |
| LimitTradeServers | 整数 | 交易服务器的最大数目。 |
| LimitWebServers | 整数 | 网络服务器的最大数目。 |
| LimitAccounts | 整数 | 交易平台中可以开立的账户的最大数目。 |
| LimitDeals | 整数 | 交易平台中可以提交的交易的最大数目。 |
| LimitGroups | 整数 | 交易平台中可以创建的组的最大数目。 |
| LiveUpdateMode | 整数 | 更新交易平台组件的模式。想要传递模式，使用EnUpdateMode枚举。 |
| TotalUsers | 整数 | 整个交易平台中的客户记录总数。 |
| TotalUsersReal | 整数 | 整个交易平台中的真实客户总数。 |
| TotalDeals | 整数 | 整个交易平台中执行的交易总数。 |
| TotalOrders | 整数 | 整个交易平台中所下的活动订单总数。 |
| TotalOrdersHistory | 整数 | 整个交易平台中的历史订单总数。 |
| TotalPositions | 整数 | 整个交易平台中的持仓总数。 |
| AccountURL | 字符串 | 客户从客户端开立演示账户时将被重定向到的地址。 |
| AccountDepositURL | 字符串 | 平台中所有账户的入金页面URL。 |
| AccountWithdrawalURL | 字符串 | 平台中所有账户的出金页面URL。 |
| AccountAuto | 整数 | 在客户端启动时自动分配模拟账户的标志：0 — 禁用，1 — 启用。 |
| AccountGroup | 对象数组 | 描述模拟账户分配组设置的对象数组。 |


用于分配模拟账户的组


| 参数 | 类型 | 描述 |
| --- | --- | --- |
| Group | 字符串 | 分配账户的组的名称。 |
| Description | 字符串 | 程序端中“账户类型”字段的描述。 |
| Company | 字符串 | 可用组所在程序端的所属公司名。 |
| Currency | 字符串 | 入金货币。 |
| CurrencyDigits | 整数 | 入金货币准确性。 |
| AccountType | 整数 | 组中的账户类型：0 — 初始账户，1 — 真实账户，2 — 模拟账户。 |
| DefaultLeverage | Float | 可用杠杆列表。 |
| Countries | 字符串 | 可供客户使用的组所在的国家列表。 |
| ConfirmationEmail | 字符串 | 邮件配置名称，通过它发送确认邮件。 |
| Agreements | 字符串数组 | 协议描述数组。 |


协议


| 参数 | 类型 | 描述 |
| --- | --- | --- |
| CaptionType | 整数 | 协议类型： 1 — 客户协议 2 — 风险免责声明 3 — 客户协议和风险免责声明 4 — 投诉处理程序 5 — 订单执行政策 6 — 客户分类流程 7 — 利益冲突条款 8 — 其他（自定义选项） |
| CaptionCustom | 字符串 | 协议名称(CaptionType = 8)。 |
| URL | 字符串 | 协议链接。 |

<a id="webapi-common-set"></a>


| | Web API / 主接口(Rest API) / 配置数据库 / 常规配置 / 设置 | |
| --- | --- | --- |






更新常规配置


该请求可以更新平台常规设置。


请求格式


> POST /api/common/set JSON格式的常规配置


响应格式


> { "retcode" : "code description", "answer" : { description } }


示例


```
//--- 发送到服务器的请求

POST /api/common/set

{

 "Name" : "Demo",

 "LiveUpdateMode" : "1",

 "AccountUrl" : "Broker",

 "AccountDepositUrl" : "broker.com",

 "AccountWithdrawalUrl" : "",

 "AccountAuto" : "0",

...

 }

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer ": {

 "Name" : "Demo",

 "Owner" : "Broker",

 "OwnerID" : "Broker",

 "OwnerHost" : "broker.com",

 "OwnerEmail" : "",

 "Product" : "MetaTrader 5",

 "ExpirationLicense" : "1577750400",

 "ExpirationSupport" : "1577750400",

 "LimitTradeServers" : "5",

 "LimitWebServers" : "5",

...

 }

}
```


请求参数


该请求没有参数。常规配置以JSON格式作为附加主体传递。只有以下参数可被修改：


- Name

- LiveUpdateMode

- AccountUrl

- AccountDepositUrl

- AccountWithdrawalUrl

- AccountAuto

- AccountGroup


服务器参数的完整描述提供在[“数据结构”](webapi_server_data_format.htm)部分。


更新时传递的配置的JSON描述与服务器返回的描述相同。


响应参数


- retcode — 如果成功，则返回响应代码 0。否则，将返回错误代码。

- answer — 以JSON格式，更新后的常规平台配置。传递参数的描述提供在“数据结构”部分。

| Balance | Float | 每日报告中的客户结余。 |
| Credit | Float | 每日报告中的客户信贷资金。 |

| AgentMonthly | Float | 当前月针对客户的交易操作收取的代理手续费的金额。 |
| BalancePrevDay | Float | 截至前一天结束的客户结余。 |

| MarginLevel | Float | 每日报告中的客户预付款水平。 |
| MarginLeverage | Float | 每日报告中的客户预付款杠杆率。 |
| 利润 | Float | 每日报告中客户所有开仓的当前利润金额。 |

| Order | 整数 | 订单的单号，因其执行了交易。 |
| Action | 整数 | 由交易执行的操作类型。以EnDealAction枚举值传递。 |
| Entry | 整数 | 交易方向。以EnDealEntry枚举值传递。 |
| Digits | 整数 | 交易价格中的小数位数。 |
| DigitsCurrency | 整数 | 已执行交易的客户入金货币的小数位数。 |
| ContractSize | Float | 交易执行所针对的交易品种的合约大小。 |
| Time | 整数 | 从 1970 年 1 月 1 日开始经过的秒计交易执行的时间。 |
| Symbol | 字符串 | 交易执行所针对的交易品种。 |
| Price | Float | 交易的价格。 |
| PriceSL | Float | 交易的“止损”价位。根据发起对应交易的订单的“止损”价位来设置进入和逆转交易的“止损”价位。平仓时对应持仓的“止损”值被用于退出交易。 |
| PriceTP | Float | 交易的“止盈”价位。根据发起这些交易的订单的“止盈”价位来设置进入和逆转交易的“止盈”价位。平仓时对应持仓的“止盈”值被用于退出交易。 |
| Volume | 整数 | 交易量。一单位相当于 1/10000 手，例如，10500表示1.05手。 |
| VolumeExt | 整数 | 精确度更高的交易量。一个单位相当于1/100000000手。 |
| Profit | Float | 来自交易的利润。 |
| Value | Float | 以客户入金货币表示的交易价值。 |
| Storage | Float | 交易的库存费大小。 |
| Commission | Float | 为交易收取的手续费金额。 |
| RateProfit | Float | 交易利润货币到客户组入金货币的汇率。 |
| RateMargin | Float | 交易预付款货币到客户的入金货币的汇率。 |
| ExpertID | 整数 | 已执行交易的策略交易系统的 ID。 |
| PositionID | 整数 | 交易的持仓识别符（单号）。 |
| Comment | 字符串 | 对交易注释。 |
| ProfitRaw | Float | 返回在交易执行所针对的交易品种的利润货币中指定。盈/损在交易执行所针对的交易品种的利润货币中表示。 |
| PricePosition | Float | 由该交易平仓的价格。 |
| VolumeClosed | 整数 | 由交易平仓的持仓量。 |
| TickValue | Float | 交易的报价价格。 |
| TickSize | Float | 交易的报价大小。 |
| Flags | 整数 | 交易的常规标识。预留该参数供将来使用。 |
| TimeMsc | 整数 | 毫秒计交易的时间。 |
| Reason | 整数 | 执行交易的原因。以EnDealReason枚举值传递。 |
| Gateway | 字符串 | 网关的 ID，使用其执行交易。 |
| PriceGateway | Float | 实际用于通过外部交易系统中的网关执行交易的价格，而不考虑其网关的价格转换设置。 |
| MarketBid | Float | 通过服务器执行交易时的市场卖价。该字段仅适用于平台更新为build 2890（或更高版本）后创建的交易。对于较早的交易，该值将为零。 |
| MarketAsk | Float | 通过服务器执行交易时的市场买价。该字段仅适用于平台更新为build 2890（或更高版本）后创建的交易。对于较早的交易，该值将为零。 |
| MarketLast | Float | 通过服务器执行交易时的市场最后价。该字段仅适用于平台更新为build 2890（或更高版本）后创建的交易。对于较早的交易，该值将为零。 |
| ModificationFlags | 整数 | 交易修改标识。在EnTradeModifyFlags枚举值（适当标识值的和）中传递。完全平仓或部分平仓的交易承得其修改标识。平仓之后，数据库中不再有单独的持仓相关条目。为了防止丢失修改相关信息，标识会被复制到平仓交易中。同时，会为交易设置其他 EnTradeModifyFlags::MODIFY_FLAGS_POSITION 修改标识，意味着标识承自持仓。 |

| | Web API / 主接口(Rest API) / 设置文件 | |
| --- | --- | --- |






设置文件


Web API可以在交易服务器上保存应用程序设置并随时进行访问。设置被分别存储在使用API的每个经理账户的特殊目录中：[trade server directory]\settings\[manager login]\.


该选项可以快速设置经理的工作位置。例如，如果经理更换了计算机，也不需要重新配置网络应用程序。相反，您可以轻松地应用之前保存在交易服务器上的设置。也可以创建几组经理设置来执行某些任务。


```
每个经理目录中只能存储一层嵌套信息，最多可以存储128个文件夹。设置文件的最大数量为1024。每个配置文件的大小不超过16 MB。
在与Web API客户端连接的服务器（连接的经理账户所在的服务器）上保存和访问设置。如果有多台服务器，则需要将设置保存在每台服务器上。
此功能对交易服务器安全可靠：Web API不可以将有潜在危险的文件类型发送到服务器（例如*.exe）。仅允许使用具有以下扩展名的文件：ini、cfg、dat、json、config、sqlite、xml、conf、settings、key、db、txt和log，以及不带扩展名的文件。此外，内容只能表示为JSON文档。


Web API可使您接收通过Manager API设置的配置文件，但前提是文件内容是JSON文档。
```


带有设置的常规操作方案


设置存储在交易服务器的特殊目录中：[trade server directory]\settings\[manager login]\。


- Web应用程序可以通过/api/setting/get请求接收设置文件。例如，在应用程序启动过程中或用户执行适当的命令时。

- 从文件中读取并应用设置。

- 如果用户在应用程序操作期间编辑设置，则应用程序可以通过/api/setting/set方法更新交易服务器上的设置文件。


请注意，设置包含一个完整的文件。您无法在服务器上更新单个设置。要更新设置，必须生成包含所有设置的整个文件。对于Web API，文件内容只能是有效的JSON文档。


不应在服务器上过于频繁地更新设置，即，每次在应用程序端编辑设置时。编辑设置后，用户可以立即更改其他设置或还原更改。建议每5分钟发送一次更新。


多个应用程序可以使用相同的设置文件，包括不同类型的API。Web API不会在服务器端追踪文件中的更改。每次调用[/api/setting/set](webapi_setting_set.htm)都会重写指定的设置文件，即使该文件已经被另一个应用程序更改过。服务器始终存储最后提交的版本。

数据结构


时间配置以JSON格式被传递为响应[/api/time/get](webapi_time_get.htm)请求。时间配置包括以下参数：


| 选项字段 | 类型 | 描述 |
| --- | --- | --- |
| Daylight | 整数 | 切换到夏令时的模式。0 — 禁用 DST 切换、1 — 启用。 |
| DaylightState | 整数 | 平台时区中包含夏令时。0 表示平台时区没有启用夏令时。否则，会使用任何非零的值。 |
| TimeZone | 整数 | 服务器的时区。从 GMT（格林威治标准时间）以分钟指定。例如: 0 = GMT; -60 = GMT - 1; 60 = GMT + 1。 |
| TimeServer | 字符串 | 同步时间的服务器的地址。 |
| Days | 整数数组 | 二维数组 [7][24]，在此第一个维度表示周内日期（从星期日开始），第二个维度表示小时。工作/非工作时间的符号以 EnTimeTableMode枚举值传递。 |
