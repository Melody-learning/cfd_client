以下命令可用于处理服务器上的客户（交易账户）：

| --- | --- |

| /client_update | 更新服务器上的客户数据。 |
| /client_delete | 从服务器删除客户。 |
| /client_get | 根据客户ID获取客户信息。 |
| /client_history_get | 获取指定时间段内客户记录变化的历史记录。 |
| /client_ids | 获取经理可用的所有客户的标识符列表。 |
| /client_user_add | 将交易账户与客户绑定。 |
| /client_user_delete | 将交易账户与客户解绑。 |
| /client_user_logins | 获取绑定客户记录的客户列表。 |
| /document_add | 将文档添加到客户记录。 |
| /document_update | 更改绑定到客户记录的文档。 |
| /document_delete | 从客户记录删除文档。 |
| /document_get | 获取客户记录文档。 |
| /document_history_get | 获取绑定到客户记录的文档的更改历史。 |
| /comment_add | 向客户记录或文档添加注释。 |
| /comment_update | 编辑客户记录或文档的注释。 |
| /comment_delete | 从客户记录或文档删除注释。 |
| /comment_get | 获取添加到文档或客户记录的注释。 |

<a id="webapi-client-add"></a>


| | Web API / 主接口(Rest API) / 客户 / 添加 | |
| --- | --- | --- |






添加客户


该请求可以创建服务器上的客户。


请求格式


> POST /api/client/add [ 一个或多个JSON格式的客户描述 ]


响应格式


> [ { "id" : "identifier", "retcode" : "code description", }, { "id" : "identifier", "retcode" : "code description", }, ... ]


示例


```
//--- 发送到服务器的请求

POST /api/client/add

[

 {

 "PersonName" :"John",

 "PersonLastName" :"Smith"

 "ClientType" :"1",

 "ClientStatus" :"700",

 "AssignedManager" :"1000"

 },

 {

 "PersonName" :"Mary",

 "PersonLastName" :"Anne"

 "ClientType" :"1",

 "ClientStatus" :"700",

 "AssignedManager" :"1200"

 }

]

//--- 服务器响应

[

 {

 "id" :"1278",

 "retcode" :"0 Done",

 },

 {

 "id" :"1279",

 "retcode" :"0 Done",

 },

 ...

]
```


请求参数


该请求没有参数。要创建的客户的描述将在请求的JSON主体中传递。


没有必填字段，任何数据集都可以传递。可能用户参数的完整描述提供在[“数据结构”](webapi_client_data_structure.htm)部分。


创建时传递的客户记录的 JSON 描述与服务器返回的描述相同。例如；


> { "ClientType" :"1", "ClientStatus" :"700", "ClientExternalID" : "", "AssignedManager" :"0", "Comment" : "automatically generated on startup", ... }


> 当创建客户时，将执行检查以确保PersonName或ContactEmail参数对于服务器上的客户数据的唯一性。


响应参数


- id — 创建的客户标识。

- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。


注意


- 客户只能添加到连接API应用程序的服务器的数据库中。

- 创建客户端时，服务器会自动为该客户端分配一个唯一标识符(RecordID）。无法手动设置标识符。

<a id="webapi-client-delete"></a>


| | Web API / 主接口(Rest API) / 客户 / 删除 | |
| --- | --- | --- |






删除客户


该请求可以从服务器删除客户。


请求格式


> GET /api/client/delete?id=list of IDs POST /api/client/delete?id=list of IDs


响应格式


> [ { "id" : "identifier", "retcode" : "code description", }, { "id" : "identifier", "retcode" : "code description", }, ... ]


示例


```
//--- 发送到服务器的请求

GET /api/client/delete?id=1278,1279

//--- 服务器响应

[

 {

 "id" : "1278",

 "retcode" : "0 Done",

 },

 {

 "id" : "1279",

 "retcode" : "0 Done",

 },

 ...

]
```


请求参数


- id — 将删除的客户ID。指定为逗号隔开的列表。


响应参数


- id — 已删除的客户ID。

- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。


注意


只能从连接到客户已被创建所在的交易服务器的应用程序中删除客户。对于所有其他应用程序，将返回响应代码[12001](retcodes_api.htm)。如果找不到该对象，则将返回响应代码[13](retcodes_common.htm)。

<a id="webapi-client-get"></a>


| | Web API / 主接口(Rest API) / 客户 / 删除 | |
| --- | --- | --- |






获取客户信息


该请求可以根据ID接收客户信息。


请求格式


> GET /client_get?id=list of IDs&group=groups POST /client_get?id=list of IDs&group=groups


响应格式


> { "retcode" : "code description", "answer" : [ { client description }, { client description }, ... ] }


示例


```
//--- 发送到服务器的请求

GET /client_get?id=1278,1279

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : [

 {

 "RecordID" : "1278",

 "PersonName" : "John",

 "PersonLastName" : "Smith"

 "ClientType" : "1",

 "ClientStatus" : "700",

 "AssignedManager" : "1000"

 ...

 },

 {

 "RecordID" : "1279",

 "PersonName" : "Mary",

 "PersonLastName" : "Anne"

 "ClientType" : "1",

 "ClientStatus" : "700",

 "AssignedManager" : "1200"

 },

 ...

 ]

}
```


请求参数


- id — 您希望获得数据的客户ID。指定为逗号隔开的列表。

- group — 通过客户指定的首选组(TradingGroup)过滤结果的可选参数。您可以指定一个组，多个组（逗号分隔）或组掩码。该掩码使用"*"（任意值）和"!"（除外）进行指定。例如："demo*,!demoforex" - 名称以'demo'开头的所有组，demoforex组除外。使用组过滤器时返回以下客户：


- 为其指定TradingGroup的客户，且该组对应请求掩码

- 没有为其指定TradingGroup的客户，但至少有一个来自请求组的绑定账户

- 没有为其指定TradingGroup的客户，且没有绑定账户（防止客户丢失）


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- answer — JSON格式的客户描述数组。传递参数的完整描述提供在“数据结构”部分。

<a id="webapi-client-history-get"></a>


| | Web API / 主接口(Rest API) / 客户 / 获取变更历史 | |
| --- | --- | --- |






获取客户变更历史


该请求可以接收指定时间段内客户记录变化的历史记录。


请求格式


```
GET /api/client/history/get?id=identifier&from=date&to=date&author=login

POST /api/client/history/get?id=identifier&from=date&to=date&author=login
```


响应格式


> { "retcode" : "code description", "answer" : [ { client state description }, { client state description }, ... ] }


示例


```
//--- 发送到服务器的请求

GET /api/client/history/get?id=1032&from=1552395829&to=1584104703

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : [

 {

 "RecordID" : "1278",

 "PersonName" : "John",

 "PersonLastName" : "Smith"

 "ClientType" : "1",

 "ClientStatus" : "700",

 "AssignedManager" : "1000"

 ...

 },

 {

 "RecordID" : "1279",

 "PersonName" : "Mary",

 "PersonLastName" : "Anne"

 "ClientType" : "2",

 "ClientStatus" : "600",

 "AssignedManager" : "1200"

 },

 ...

 ]

}
```


请求参数


- id — 您希望获得变更历史记录的客户ID。

- from — 您希望获得客户变更历史的时段的开始。该日期从1970.01.01开始，以秒指定。

- to — 您希望获得客户变更历史的时段的结束。该日期从1970.01.01开始，以秒指定。

- author — 更改客户的经理账户登录名。它被用作过滤器。可选参数。如果没有指定作者或者设置为零，则返回任何经历所做的更改。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- answer — JSON格式的客户状态数组。传递参数的完整描述提供在“数据结构”部分。


注意


该请求返回客户描述数组：指定作者在指定的时间段内更改后的所有客户状态。

获取备份列表

请求格式


```
GET /api/user/backup/list?from=beginning&to=end&server=identifier

POST /api/user/backup/list?from=beginning&to=end&server=identifier
```


响应格式


> { "retcode" : "code description", "answer" : [ date list ] }

```
//--- 发送到服务器的请求

GET /api/user/backup/list?from=1546357749&to=1574351349
```

<a id="webapi-user-balance-check"></a>


| | Web API / 主接口(Rest API) / 用户 / 检查结余 | |
| --- | --- | --- |






检查用户结余


该请求可以使您检查是否需要调整用户结余和信贷资金。


请求格式


```
GET /api/user/check_balance?login=login&fixflag=flag

POST /api/user/check_balance?login=login&fixflag=flag
```


响应格式


```
{

 "retcode" : "code description",

 "answer" : {

 "Balance": { "User" : "New value", "History" : "Old value" },

 "Credit": { "User" : "New value", "History" : "Old value" }

}
```


示例


```
//--- 发送到服务器的请求

GET /api/user/check_balance?login=611129&fixflag=1

//--- 服务器响应

{

 "retcode" : "0 Done",

 "answer" : {

 "Balance": { "User" : "3000.000000", "History" : "2495.000000" },

 "Credit": { "User" : "100.000000", "History" : "96.000000" }

}
```


请求参数


- login — 您希望检查其结余的用户的登录名。

- fixflag — 检查后需要更正客户结余和信贷资金的标识。如果fixflag等于1，则客户的结余和信贷资金根据交易历史进行调整。如果该标志为0，则不会进行更正。


响应参数


- retcode — 如果成功，该命令返回响应代码 0。否则，将返回错误代码。

- balance — 更正后（用户）和更正前（历史）的用户结余。

- credit — 更正后（用户）和更正前（历史）的用户信贷资金。


注意


该请求根据用户的交易历史来检查其结余，并在必要时对用户结余进行更正。信贷资金根据[IMTDeal::Credit](imtdeal_enum.htm#endealaction)类型交易进行检查。

- password — 需要检查的密码。出于安全原因，我们建议在附加正文中而不是在请求参数中发送密码。


> 可以在命令参数中、在JSON格式的附加正文中或者两者同时传递客户描述。在附加正文中传递的描述具有更高的优先级。 我们强烈建议您不要在命令参数中传递密码，因为请求地址可能会被请求从客户端到服务器所经过的中间网络设备记录/缓存。始终在额外请求正文中发送密码。


响应参数


- retcode — 如果成功，命令返回响应代码 0。否则将返回错误代码。例如，代码 3006表示密码不正确。

<a id="webapi-user-update"></a>


| | Web API / 主接口(Rest API) / 用户 / 更新 | |
| --- | --- | --- |






用户更新


该请求可以更改交易服务器上的用户账户记录。


请求格式


```
GET /api/user/update?login=login&rights=rights&group=group&name=name&company=company&language=language&city=city&state=state&zipcode=zipcode&address=address&phone=phone&

email=email&id=id&status=status&comment=comment&color=color&pass_phone=password&leverage=leverage&agent=account\r\n



POST /api/user/update?login=login&rights=rights&group=group&name=name&company=company&language=language&city=city&state=state&zipcode=zipcode&address=address&phone=phone&

email=email&id=id&status=status&comment=comment&color=color&pass_phone=password&leverage=leverage&agent=account\r\n

{ JSON格式的将要创建的用户描述 }
```


响应格式


> { "retcode" :"0 Done", "answer" : { description } }


示例


```
//--- 发送到服务器的请求

POST /api/user/update?login=954402&group=demoforex&name=JohnSmith&leverage=200

{

 "MQID" :"5CD369A9",

 "Company" :"Individual",

 "Country" :"United States",

 "City" :"New York"

}



//--- 服务器响应

{

 "retcode" :"0 Done",

 "answer" : {

 "Login" :"954402",

 "Group" : "demoforex",

 "CertSerialNumber" :"0",

 "Rights" :"2531",

 "MQID" :"5CD369A9",

 "Registration" :"1572956466",

 "LastAccess" :"1572956466",

 "LastPassChange" :"1572956466",

 "Name" :"JohnSmith",

 "Company" :"Individual",

...

 }

}
```


请求参数


- login — 应该更新的账户的登录名。该字段必填。

- rights — 用户权限。使用EnUserRights 枚举的值传递（适当标识的值的和）。

- group — 应该将账户移动到该组。

- name — 客户的名字。客户交易品种名称的最大长度是 128 个字符（包括行结束字符）。较长的字符串将被缩减到这个长度。

- company — 客户公司的名称。公司名称的最大长度是 64 个字符（包括行结束字符）。较长的字符串将被缩减到这个长度。

- language — 客户的语言。在 LangID格式中指定的 MS Windows 系统（从Prim.Lang.标识符的值）

- country — 用户的居住国家。

- city — 用户的居住城市。

- state — 用户的居住州（地区）。

- zipcode — 用户的邮政编码。

- address — 用户的地址。地址最大长度限于128个字符（包括行结束符）。较长的字符串将被缩减到这个长度。

- phone — 用户的电话号码。

- email — 用户的电子邮件地址。

- id — 客户的身份证件号码。

- status — 客户的状态。

- comment — 用户的注释，注释长度限于64个字符（包括行结束字符）。较长的字符串将被缩减到这个长度。

- color — 通过经理端处理请求时显示的客户请求的颜色。

- pass_phone or PhonePassword — 用户的电话密码。

- leverage — 从1至500的杠杆中客户杠杆的大小。杠杆不能等于0。

- account — 外部银行中用户账户的数量。对应于MetaTrader 5管理端中的“银行账户”字段。

- agent — 代理账户的数量。


命令参数之后，您可以指定JSON格式的用户账户记录的已更改参数描述。在其他描述中传递的用户记录的参数覆写在命令的正文中传递的参数。可能用户参数的完整描述提供在[“数据结构”](webapi_user_data_structure.htm)部分。


更新时传递的用户记录的JSON 描述与服务器返回的描述相同。例如：


```
{

"Login" :"1023",

"Group" : "demo\demoforex",

"CertSerialNumber" :"0",

"Rights" :"483",

"Registration" :"1314700797",

"LastAccess" :"1314700797",

"LastIP" :"0.0.0.0",

"Name" :"John Smith",

...

}
```


> 如果参数值不正确，则参数不会更改。 不能从Web API更改的用户的参数（例如，CertSerialNumber）也不会更改。 可以在命令参数中、在JSON格式的附加主体中或者两者同时传递客户描述。


响应参数


- retcode — 如果成功，命令返回响应代码 0。否则，其将返回错误代码。

- answer — JSON格式的已更新用户参数。传递客户参数的完整描述提供在“数据结构”部分。


注意


仅在您被连接至账户所在的同一交易服务器时才可以更新用户账户。如果未找到具有指定登录的帐户，则返回代码[13](retcodes_common.htm)。


请注意[ApiData属性](webapi_user_data_structure.htm#apidata)使用详情。每个属性单元格可接受的字段（始终为16）：


- 必需字段AppID和ID，包含应用程序ID和属性ID。

- 其中一个字段：ValueInt、ValueUInt或ValueDouble，以确定单元格值类型。如果指定多个字段，将按照以下顺序检查字段是否存在：ValueInt、ValueUInt、 ValueDouble。在这种情况下，将使用发现的最后一个字段类型。例如，要写入'unsigned'类型的值，请指定：


> { ... "ApiData": [ { "AppID":"1", "ID":"2", "ValueUInt":"3", }, ... ], ... }


相应地，当使用*_get_*方法时，从ApiData数学中相应单元格的ValueUInt字段获取值。
